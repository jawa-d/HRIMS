تقرير آلية عمل النظام ومدى امتيازه

اسم النظام: HRMS - Human Resources Management System
النسخة المرجعية: main
التاريخ: 8 مارس 2026

الملخص التنفيذي

النظام مبني على HTML/CSS/Vanilla JS مع Firebase Authentication وFirestore. يطبق صلاحيات حسب الدور ويغطي الموارد البشرية والرواتب والحسابات بشكل متكامل. مستوى الجاهزية مرتفع للتشغيل الفعلي.

آلية عمل النظام

  1. تسجيل الدخول والتحقق من الهوية والصلاحية حسب الدور.
  2. تحميل الواجهة والوحدات حسب الصلاحيات (Sidebar/Controllers).
  3. قراءة/كتابة البيانات عبر طبقة الخدمات إلى Firestore.
  4. تطبيق سير العمليات: موظفين، حضور، إجازات، رواتب، محاسبة، تقارير.
  5. تتبع أحداث الاستخدام وإشعارات النظام.

الوحدات الرئيسية

  • الموظفين، الأقسام، المناصب
  • الحضور والإجازات
  • الرواتب (حفظ مسودة، اعتماد، تصدير CSV/PDF)
  • الحسابات والتدفق النقدي والإدارة المحاسبية
  • التقارير والإشعارات والإعدادات

نقاط الامتياز

  • Role-Based Access Control على مستوى الصفحة والإجراء.
  • فصل واضح بين واجهة المستخدم ومنطق الصفحات وطبقة الخدمات.
  • دعم عربي/إنجليزي وRTL/LTR.
  • تقارير وتصدير متعدد (CSV/Excel/PDF).
  • تكامل HR + Finance ضمن منصة واحدة.

الجاهزية التشغيلية

التقييم: 8.8 / 10 - جاهز بدرجة عالية للتشغيل الحقيقي.

تم التحقق من ملفات JavaScript، ومعالجة مشكلة تصدير في الحسابات، ورفع التحديثات على الفرع الرئيسي.

توصيات قبل الإطلاق

  1. اختبار نهائي سريع على صفحات payroll وaccounting وaccounting_admin وaccounting_flow.
  2. اختبار صلاحيات الأدوار الأربعة بحسابات حقيقية.
  3. مراجعة قواعد Firestore وتأكيد النسخ الاحتياطي الدوري.
